PulumiCost can be deployed in various environments, from local developer machines to CI/CD pipelines and containerized setups.

## Deployment Options

### Local Development

For individual developers, PulumiCost runs as a standalone CLI tool or as a Pulumi Analyzer plugin.

- [Installation Guide](../getting-started/installation.md)
- [Analyzer Setup](../getting-started/analyzer-setup.md)

### CI/CD Pipelines

Integrate PulumiCost into your GitHub Actions, GitLab CI, or Jenkins pipelines to enforce cost policies.

- [CI/CD Integration](cicd-integration.md)

### Docker Containers

Run PulumiCost in a consistent, isolated environment using our official Docker images.

- [Docker Guide](docker.md)

## Enterprise Deployment

For large-scale deployments, consider:

- Centralized configuration management
- Shared plugin registries
- Custom policy packs distributed via version control
